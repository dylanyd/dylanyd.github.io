Hardware and software architecture in Autonomous vehicles

Sensors for perception

  • Exteroceptive Sensors for sensing surroundings
    • Camera (Metrics: Resolution, Field of view, Dynamic range)
    • Stereo camera (combination of two cameras): enables depth estimation from image data
    • LIDAR: 3D scene geometry (Metrics: Number of beams, Points per second, Rotation rate, Field of view)
    • Radar: robustly detect large objects in the environment and useful in adverse weather. (Metrics: Range, Field of view, Position and speed accuracy)
    • Ultrasonics: for parking
  • Proprioceptive sensors for sensing internal
    • Global Navigation Satellite Systems (GNSS) / IMU: measure ego vehicle position, velocity, and sometimes heading
    • wheel odometry sensors: track the wheel rates of rotation, and uses these to estimate the speed and heading rate of change of the ego car.

Sensor configuration

For different scenarios, requirements of sensor coverage are different. It is important to ensure there is no blind spots in overall coverage of perception.

  • Longitudinal coverage: Ego-vehicle should perceive objects in the current lane whatever in the front (for vehicle following, emergency stop, and overtaking) or behind (for lane change).
  • Lateral coverage: Ego-vehicle should perceive objects in adjacent and further lanes (for lane change, emergency stop, overtaking, turning at intersections, and pass roundabouts).
  • For different tasks, the perception ranges are different.

Software architecture

High level

Perception Module

Perception module involves perceive environmental information (localization of ego-vehicle; detect static objects; detect and track dynamic objects; predict the motion of dynamic objects) and create/update environment map (Occupancy Grid Map; Localization Map; Detailed Road Map) using the information.

Planning module

  • Mission planner: Create a mission path from current position to current goal
  • Behavior planner: Output current behavior (e.g. lane change; speed up)
  • Local planner: Output current running trajectory

Controller module

System supervisor

System supervisor monitors whether the whole autonomous driving system works normally.

  • Hardware Supervisor:
    • Monitoring statuses of all hardware (e.g., broken of camera).
    • Monitoring sensors’ outputs (e.g., corrupting LIDAR point cloud data).
  • Software Supervisor:
    • Validating this software stack to make sure that all elements are running as intended at the right frequencies and providing complete outputs.
    • Analyzing inconsistencies between the outputs of all modules.
